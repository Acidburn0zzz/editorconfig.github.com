EditorConfig

What is EditorConfig?

EditorConfig is an initiative for solving the my indentation style is different than yours problem common amongst open source projects. An EditorConfig file is used to define the recommended indentation styles for a group of files. The goal of this project is to finalize the EditorConfig file format and create plugins to add EditorConfig support to popular text editors and IDEs.

What's an EditorConfig file look like?

The EditorConfig file format is not yet finalized. This is the definition for the current editorconfig version pending finalization (v0.2).

Example file

The below file specifies that all files with a py file extension (Python files) use 4 spaces for indentation and all files with a js file extension (JavaScript files) use tabs for indentation.

[*.py]
indent_style = space
tab_width = 4

[*.js]
indent_style = tab

Where are these files stored?

When a file is opened in a text editor with an EditorConfig plugin, a search is performed in the current directory and all parent directories for a file called .editorconfig. All found .editorconfig files are searched for sections with filename globs matching the current filename. If multiple .editorconfig files have matching globs, the rules from the closer .editorconfig file takes precedence.

In the above example, editors should use 4 spaces for indenting when editing the Python files in the project and hard tabs for indenting in JavaScript files.

Check out the EditorConfig file for this website which defines 2 spaces for indentation of HTML and CSS files.

Details of the file format

EditorConfig files use an INI file format in which sections names are globs matching filenames in the format accepted by the fnmatch C library. Comments can be added to these files by prepending a semicolon (;) to the beginning of a line. Comments and blank lines are ignored when parsing these files.

Wildcard patterns

Section names in EditorConfig files are file globs. These file globs support pattern matching through Unix shell-style wildcards. The following special characters are used in these shell-style wildcards:

*Matches any string of characters, except for path separators (/ character)
?Matches any single character
[seq]Matches any single character in seq
[!seq]Matches any single character not in seq

The backslash character (\) can be used to escape a character so it is not interpreted as a special character.

Supported properties

EditorConfig file sections contain properties, which are name-value pairs separated by an equal sign (=). Property names can contain letters, numbers, and underscores. EditorConfig plugins will ignore any unrecognized property names and any properties with invalid values.

Here is the list of all property names understood by EditorConfig and all valid values for these properties:

  • indent_style: can be set to tab or space to use hard tabs or spaces respectively.
  • tab_width: if indent_style is space, this should be a whole number defining the number of spaces used for each indentation level.

Supported text editors

The EditorConfig file format is not finalized yet, but there's a vim plugin to demonstrate and test the format so far. You can find the editorconfig vim plugin here. Eventually the file format will be finalized and plugins will be written for other popular text editors.

Contributing to EditorConfig

Give us your feedback

This project is greatly in need of feedback from other developers. We want to hear ideas about how to make this project better. Please use the mailing list to send an email to the editorconfig team and use the issue tracker to submit bugs.

Create a plugin

EditorConfig plugins can be developed by using the editorconfig core code found in the main editorconfig repository. The editorconfig core is a C program that accepts as input the file being edited, finds and parses applicable .editorconfig files, and passes back the options that should be used.

If you are planning on creating a new plugin, use the mailing list to let us know so we can help out and link to your plugin once it's online. If you plan on using the EditorConfig core as a C library, the documentation may be helpful.
